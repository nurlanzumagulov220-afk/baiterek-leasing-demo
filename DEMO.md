# Сценарий живого демо

## Подготовка (за 5 минут до)

```bash
# Терминал 1
cd aksakal && go run aksakal.go

# Терминал 2
cd backend && .venv/bin/uvicorn main:app --reload --port 8000

# Терминал 3
cd frontend && npm run dev
```

Открыть в браузере: **http://localhost:5173**

---

## Сценарий 1 — «Шеф создаёт меню» (2 мин)

**Заходим в Admin → Конструктор форм**

Показываем атом `leasing_wagons_s1.json` в редакторе — это весь продукт в одном файле.
Меняем ставку прямо в UI:
```json
"rate_annual_pct": 14.0
```
Жмём «Сохранить». Говорим: *«Ставка изменилась для всех 50 000 предпринимателей Казахстана. Прямо сейчас. Без единой строки кода.»*

**curl для жюри:**
```bash
curl http://localhost:8000/atoms/leasing_wagons_s1 | python3 -m json.tool | grep -A5 '"params"'
```

## Сценарий 2 — «Гость делает хороший заказ» (2 мин)

Регистрируемся как предприниматель. Идём в Услуги → Лизинг вагонов I этап.

- Шаг 1: вводим БИН `123456789012`, жмём «📡 eGov» — поля заполняются автоматически
- Шаг 2: вводим сумму `50 000 000`, аванс `20%`, срок `36 мес` — калькулятор считает в реальном времени
- Шаг 3: прикладываем файлы
- Шаг 4: подаём

**Результат:** зелёный экран с `trust_score: 87%` и `tamga_id`.

**curl:**
```bash
curl -X POST http://localhost:8000/applications/leasing_wagons_s1 \
  -H "Content-Type: application/json" \
  -d '{"form_data":{"bin":"123456789012","company_name":"ТОО Тест","director_name":"Иванов","phone":"+7 777 123-45-67","email":"t@t.kz","region":"Алматы","equipment_type":"Вагоны грузовые","equipment_count":"5","equipment_cost":"50000000","advance_pct":"20","term_months":"36","consent_personal_data":true,"consent_accuracy":true}}'
```

## Сценарий 3 — «Система блокирует плохой заказ» (1 мин)

Та же форма. На шаге 3 «Документы» — **ничего не загружаем**. Жмём «Подать заявку».

**Результат:** красный экран. Aksakal заблокировал:
- `Документы не приложены`
- `Заполнено только 81% обязательных полей`
- Trust Score: `76%` — ниже порога 80%

*«Заявка даже не попала в базу данных. Её нет нигде.»*

## Сценарий 4 — «Администратор подписывает чек» (1 мин)

Admin → Заявки → выбираем одобренную заявку → меняем статус на `Одобрено`.

```bash
curl -X PATCH http://localhost:8000/applications/{APP_ID}/status \
  -H "Authorization: Bearer {TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"status":"approved"}'
```

В ответе:
```json
"verification": {
  "amanat_id": "7af6a399...",
  "amanat_signature": "a3f1b2c4..."
}
```

*«Aksakal подписал график платежей своим закрытым ключом Ed25519. Изменить задним числом — невозможно.»*

## Сценарий 5 — «Спросить Аксакала» (1 мин)

Admin → панель «Спросить Аксакала» → вводим:

- `«Есть ли подозрительные заявки?»`
- `«Сколько заявок одобрено?»`
- `«Как обучена модель?»`

**curl:**
```bash
curl -X POST http://localhost:8000/admin/ask-aksakal \
  -H "Authorization: Bearer {ADMIN_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"question":"есть ли подозрительные заявки?"}'
```

## Контрольные вопросы жюри

**«Это hardcode?»**
> Нет. Измените `rate_annual_pct` в JSON — ставка изменится немедленно. Создайте новый atom_id — новая услуга появится в каталоге без единой строки кода.

**«Данные утекают в облако?»**
> Нет. Aksakal работает на сервере Байтерека. Федеративное обучение передаёт только математические градиенты, не сырые данные заявок. Соответствует п. 5 ТЗ.

**«Как масштабируется на 70+ услуг?»**
> Каждая услуга — новый JSON-файл. Форма, валидация, калькулятор, верификация — всё наследуется автоматически.
